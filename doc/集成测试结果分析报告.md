## 三、测试架构总结

### Mock 分层策略

```
Layer 1: sqlmock              → MySQL DAO 测试 (75.4%)
Layer 2: miniredis            → Redis DAO 测试 (81.6%)
Layer 3: Docker Kafka          → 生产者/消费者 E2E (52.5%)
Layer 4: Docker Minio          → 文件上传集成测试 (70.8%)
Layer 5: Docker Canal + Kafka  → 缓存一致性 E2E (验证通过)
```

### 关键修复记录

| Bug | 文件 | 修复 |
|:---|:---|:---|
| MD5加盐算法错误 | `dao/mysql/user.go:31-34` | `h.Write(secret); h.Write(password); h.Sum(nil)` |
| Kafka 29092端口未暴露 | `docker-compose.yml` | 添加 `"29092:29092"` 端口映射 |
| Canal密码错误 | `docker-compose.yml` | password→123456 |
| Canal v1.1.5 segfault | `docker-compose.yml` | 升级到 v1.1.7 |
| sqlmock正则贪婪匹配 | 多处 | `(.+)` → `.*` |
| Kafka GroupID冲突 | E2E测试 | 使用时间戳唯一 GroupID |
